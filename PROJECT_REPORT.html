Abstract

The AI Resume Analyzer is a web-based application designed to help job seekers and recruiters by automatically parsing, analyzing, and evaluating resumes using Natural Language Processing (NLP) techniques. The system extracts key information from PDF resumes including personal details, skills, education, and experience, then provides intelligent recommendations for skill enhancement, career path predictions, and relevant courses.

Built using Python with the Streamlit framework, the application leverages the spaCy NLP library for Named Entity Recognition (NER), pdfminer3 for PDF text extraction, and Plotly for interactive data visualization. The system implements a rule-based keyword matching algorithm to predict job roles across five major domains: Data Science, Web Development, Android Development, iOS Development, and UI/UX Design.

Key features include individual resume analysis with detailed scoring (out of 100), bulk resume processing for recruiters, course and certification recommendations, interview preparation resources, and a comprehensive admin dashboard with 8 interactive analytical visualizations including donut charts, bar charts, and histograms. The enhanced feedback system features real-time statistics, rating distribution analysis, and a styled comments feed. The application stores all data in a SQLite database, enabling organizations to track applicant trends, generate CSV reports, and make data-driven hiring decisions.

Keywords: Natural Language Processing, Resume Parsing, Named Entity Recognition, Skill Recommendation, Job Role Prediction, Streamlit, spaCy, Machine Learning

1.8 Proposed Methodology

The AI Resume Analyzer employs a multi-stage pipeline approach for comprehensive resume analysis:

1.8.1 Data Acquisition Layer

The system accepts PDF resumes through a web-based upload interface. Files are validated for format compliance, saved to the server, and queued for processing. The pdfminer3 library extracts raw text from PDF pages, handling various layouts and formatting styles.

1.8.2 Natural Language Processing Layer

Extracted text undergoes NLP processing using the spaCy library's English language model (en_core_web_sm). This layer performs:

  • Tokenization: Breaking text into individual tokens (words, punctuation)
  • Named Entity Recognition (NER): Identifying and classifying entities (PERSON, ORG, GPE, DATE)
  • Part-of-Speech Tagging: Understanding grammatical structure for context
  • Dependency Parsing: Analyzing relationships between words

1.8.3 Analysis & Prediction Layer

The structured data from NLP processing feeds into the analysis engine, which performs:

  • Experience Classification: Rule-based detection of Fresher/Intermediate/Experienced levels
  • Field Prediction: Keyword matching against domain-specific skill taxonomies
  • Resume Scoring: Weighted evaluation of resume completeness across 10 criteria

1.8.4 Recommendation & Visualization Layer

Based on analysis results, the system generates personalized recommendations and presents them through an interactive dashboard with Plotly visualizations, Streamlit components, and styled HTML/CSS elements.

2.7 NLP Techniques in Resume Parsing

A detailed review of the key NLP techniques employed in resume analysis systems:

2.7.1 Named Entity Recognition (NER)

NER is the cornerstone of resume parsing, responsible for identifying and classifying named entities in unstructured text. In the context of resume analysis, NER performs the following tasks:

2.7.2 Text Classification

Text classification techniques are used to categorize resume sections and predict job domains. Common approaches include:

  • Rule-Based Classification: Using predefined keyword lists to match against resume content. Fast and interpretable, but limited in handling synonyms and context.
  • TF-IDF + Cosine Similarity: Converting text to numerical vectors and computing similarity with job descriptions. More flexible but requires reference documents.
  • Machine Learning Classifiers: Training SVM, Random Forest, or Naive Bayes classifiers on labeled resume datasets. More accurate but requires labeled training data.
  • Deep Learning: Using BERT, transformer models for semantic understanding. Highest accuracy but computationally expensive and requires large datasets.

2.7.3 Information Extraction Pipeline

A typical resume parsing pipeline involves the following stages:

  1. Document Preprocessing: PDF to text conversion using libraries like pdfminer3, PyPDF2, or pdfplumber
  2. Text Cleaning: Removal of special characters, extra whitespace, and formatting artifacts
  3. Section Detection: Identifying resume sections (Education, Experience, Skills) using header keywords and formatting patterns
  4. Entity Extraction: Running NER model on each section to extract structured information
  5. Skill Matching: Comparing extracted skills against predefined skill taxonomies or ontologies
  6. Post-Processing: Validation, normalization, and deduplication of extracted entities

2.8 Resume Analysis Algorithms

2.8.1 Keyword Matching Algorithm

The most common approach in resume analysis involves matching extracted skills and keywords against predefined lists. This project uses an enhanced keyword matching algorithm that:

  • Converts both resume text and keyword lists to lowercase for case-insensitive matching
  • Uses word boundary detection to avoid partial matches (e.g., "java" vs "javascript")
  • Counts matching keywords per domain and selects the domain with highest count
  • Provides a confidence score based on the number of matching keywords

6.3 Product Perspective

The AI Resume Analyzer is a standalone web application that operates independently. It interfaces with:

  • File System: For storing uploaded PDF resumes
  • SQLite Database: For persistent data storage
  • NLP Libraries: spaCy and NLTK for text processing
  • Geolocation APIs: For user location detection (optional)
  • Web Browser: As the client interface via Streamlit

6.6 Constraints

  • File Format: Only PDF format is currently supported
  • File Size: Maximum 200MB upload size (Streamlit default limit)
  • Language: English language resumes only (spaCy en_core_web_sm model)
  • Domains: Five predefined career domains for prediction
  • Network: Geolocation features require internet connectivity (optional)
  • Storage: SQLite limits concurrent write operations
  • Browser: Requires modern browser with JavaScript enabled

6.7 Assumptions and Dependencies

6.7.1 Assumptions

  • Users will upload legitimate, well-formatted PDF resumes
  • Users have basic familiarity with web applications
  • The server has sufficient storage for uploaded resumes
  • spaCy English language model is pre-installed
  • Administrators will maintain the confidentiality of credentials

6.7.2 Dependencies

6.8.2 Performance Requirements

  • Resume parsing must complete within 5 seconds for standard PDFs
  • Admin dashboard must load within 10 seconds with up to 1000 user records
  • CSV export must handle up to 10,000 records
  • Bulk upload must process resumes sequentially with progress indication
  • System must support at least 5 concurrent users

6.8.3 Security Requirements

  • Admin credentials must be required for dashboard access
  • User data must not be accessible to other users
  • Uploaded resumes must be stored in a designated directory
  • No user data shall be transmitted to external servers
  • Database must be protected from SQL injection attacks

# Geolocation using IP address
import geocoder
from geopy.geocoders import Nominatim

def get_user_location():
    try:
        g = geocoder.ip('me')
        if g.ok:
            lat, lng = g.latlng
            geolocator = Nominatim(user_agent="resume-analyzer")
            location = geolocator.reverse(f"{lat}, {lng}")
            address = location.raw['address']
            return {
                'city': address.get('city', 'Unknown'),
                'state': address.get('state', 'Unknown'),
                'country': address.get('country', 'Unknown')
            }
    except Exception:
        # Graceful fallback
        return {'city': 'Unknown', 'state': 'Unknown', 'country': 'Unknown'}
